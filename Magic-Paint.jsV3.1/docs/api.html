MOD API

window.AnimationApp が Magic Paint の公開APIです。MOD の main.js 内で const api = window.AnimationApp; として参照し、図形・ツール・UIなどを自由に追加できます。このページではMODのファイル構成から全APIリファレンスまでをまとめて説明します。

MODとは

MOD（モッド）は、Magic Paint 本体を書き換えずに機能を追加できる小さな拡張パッケージです。mods/ フォルダに置いた JavaScript / JSON / CSS が起動時に読み込まれ、window.AnimationApp という共通APIを通じてエディタと連携します。

ゲームのMODと同じ考え方で、有効/無効の切り替えや複数MODの併用ができ、保存したプロジェクト（.mlc）にも図形データとして残ります。

カスタム図形

独自の描画ロジックを持つ図形タイプを追加できます。

カスタムツール

右パネルにツールボタンを追加し、マウス操作を乗っ取れます。

カスタムブラシ

ストローク描画を完全にコントロールするブラシを追加できます。

カスタムUI

top / right / bottom / left / editor-top エリアにHTMLパネルを追加できます。

ファイルメニュー拡張

「File」メニューに独自の書き出し・読み込み項目を追加できます。

Three.js統合

WebGLレンダラーを2Dレイヤーと同じ管理下（表示/ロック/順序）に置けます。

ファイル構成

MODは mods/ 以下に、MODのIDと同じ名前のフォルダとして配置します。最小構成は次の3ファイルです。

mods/
└─ my_mod/ ← MODのIDと同じ名前にする
    ├─ mod.json ← 必須。メタデータと読み込むファイルのリスト
    ├─ main.js ← 必須。エントリポイント
    └─ style.css ← オプション
MOD IDに使えるのはアルファベット・数字・アンダースコアのみです。フォルダ名・mod.json の id・registerMod() に渡す id はすべて一致させてください。

mod.json

MODのメタデータと、読み込むファイルの一覧を書くマニフェストです。Magic Paint はこのファイルを読んでMOD一覧に表示します。

{
  "id": "my_mod",
  "name": "My MOD",
  "version": "1.0.0",
  "level": 1,
  "enabled": true,
  "description": "説明",
  "scripts": ["main.js"],
  "styles": []
}
フィールド型説明
idstringシステム内でユニークな識別子。mods/ 内のフォルダ名と一致させる
namestringModsパネルや設定画面に表示される名前
versionstringセマンティックバージョン（例: 1.0.0）
levelnumber権限レベル。将来の権限管理用フィールドで、現在は 1 固定
enabledbooleanfalse にするとサーバーがこのMODを読み込まない
descriptionstringModsパネルに表示される説明文
scriptsstring[]読み込むJSファイルのリスト（配列の順番どおりに実行される）
stylesstring[]読み込むCSSファイルのリスト（省略可）

main.js

MODの実処理を書くエントリポイントです。即時関数で囲んでグローバル変数の汚染を防ぎ、まず api.registerMod() を呼んでシステムに存在を知らせるのが基本形です。

(function () {
  const api = window.AnimationApp;
  if (!api) return;          // Magic Paint 以外の環境対策

  api.registerMod({
    id: "my_mod",
    name: "My MOD",
    version: "1.0.0",
    description: "説明"
  });

  // ここから機能を追加する（registerShapeType / registerTool など）
})();
if (!api) return; を入れておくと、Magic Paint 以外の環境で誤って読み込まれてもエラーになりません。

読み込みの仕組み

  1. Magic Paint 起動時に /api/mods からMOD一覧を取得します。
  2. enabled: true のMODについて、scripts に書かれたJSファイルを <script> タグとして動的に読み込みます。
  3. 各 main.js が実行され、内部で api.registerMod() が呼ばれた時点でそのMODがシステムに登録されます。

MODの有効/無効は、上部ツールバーの「Mods」ボタンから切り替えられます。無効化してもファイルは削除されません。

保存データへの影響

addShape() で追加した図形は、通常の図形と同じように .mlc プロジェクトファイルへ保存されます。

再読み込み時にそのMODが無効化されていると、図形タイプが未登録のまま読み込まれます（データ自体は失われません）。保存データにはMODの識別子が記録されるため、必要なMODが不足している場合はMagic Paintが警告を表示します。

クイックスタート

実際に手を動かして最初のMODを作る手順です。完成例は mods/sample_shape/ を参照してください。

1

フォルダを作る

mods/my_mod/ のように、MOD IDと同じ名前のフォルダを作成し、中に mod.json と main.js を用意します。

2

mod.json を書く

上記の最小構成をコピーし、id / name / description を自分のMODに合わせて書き換えます。

3

main.js に registerMod() を書く

即時関数の先頭で api.registerMod() を呼びます。これを呼ばないとMODがシステムに認識されません。

4

図形・ツールを登録する

registerShapeType() → registerTool() の順で呼ぶのが安全です。両方とも registerMod() の後に呼んでください。複数の図形タイプ・複数のツールを登録することもできます。

5

マウス操作を実装する

キャンバス要素にイベントリスナーを追加し、api.activeModTool?.id で自分のツールが選択中か確認してから処理します（下記「よくあるパターン」も参照）。

6

Magic Paintで有効化する

Magic Paintを起動し、上部バーの「Mods」ボタンをクリック。作成したMODの名前が一覧に出ていれば認識されています。有効化して「適用」をクリックすると右パネルにボタンが追加されます。

認識されない場合は、ブラウザで /api/mods を開いてレスポンスを確認してください。mod.json のJSONが壊れているとサーバーが無視することがあります。

APIリファレンス

ここからは window.AnimationApp（api）が提供する全メソッドを一つずつ説明します。まずは const api = window.AnimationApp; で参照を取得してください。

registerMod registerTool registerBrush registerShapeType addShape getSelected setSelectedPatch redraw registerUI registerFileMenuItem createLayer getLayers addObject registerThreeRenderer toast / setStatus getObjectsBy*

基本

全てのAPI呼び出しの起点になる参照です。main.js の一番上で取得してください。Magic Paint 以外の環境（例: テスト用の素のブラウザ）では undefined になるため、if (!api) return; で防御するのが安全です。

const api = window.AnimationApp;

登録系

registerMod(mod)

MODをシステムに登録します。main.jsの先頭で必ず呼んでください。これを呼ぶまでMagic Paintはこのスクリプトを「MOD」として認識しません。id・name・version・description はmod.jsonの内容と一致させておくと管理が楽になります。

api.registerMod({
  id: "my_mod",          // 必須。ユニークな識別子
  name: "My MOD",        // 表示名
  version: "1.0.0",
  description: "説明文"
});

registerTool(tool)

右パネルの「操作」エリアにツールボタンを追加します。ユーザーがボタンを押すとそのツールがアクティブになり、以後のキャンバス操作をMOD側で乗っ取れるようになります。

api.registerTool({
  id: "my_tool",         // 必須
  name: "マイツール",    // ボタンの title に表示
  icon: "◆"             // ボタン内 HTML（絵文字・HTMLどちらも可）
});

自分のツールが現在アクティブかどうかは、マウスイベントのハンドラ内で次のように確認します。アクティブでない場合は何もせず抜けるのが基本パターンです。

if (api.activeModTool?.id === "my_tool") { ... }

registerBrush(brush)

カスタムブラシを登録します。onStart / onMove / onEnd の3つのコールバックで、ストロークの開始・移動・終了それぞれのCanvas描画を自分でコントロールできます。

api.registerBrush({
  id: "my_brush",
  name: "マイブラシ",
  icon: "🖌",
  onStart(ctx, point, shape) { ... },
  onMove(ctx, point, shape)  { ... },
  onEnd(ctx, point, shape)   { ... }
});

point は { x, y, pressure } の形で渡されます。pressure は筆圧対応デバイスでの筆圧値（0〜1、非対応デバイスでは1）です。

registerShapeType(type, renderer)

独自の図形タイプを登録します。図形1つぶんの「描画・当たり判定・移動・リサイズ・書き出し」をまとめて定義するAPIで、MODの中心となる機能です。draw だけが必須で、他は用途に応じて省略できます。

api.registerShapeType("my_shape", {
  // 必須: Canvas 2D で図形を描画する
  draw(ctx, s) {
    ctx.save();
    ctx.globalAlpha = (s.opa || 100) / 100;
    ctx.strokeStyle = s.color || "#fff";
    ctx.lineWidth = s.sw || 2;
    // ... 描画処理 ...
    ctx.restore();
  },

  // バウンディングボックスを返す
  getBounds(s) {
    return { x: s.x, y: s.y, w: s.w, h: s.h };
  },

  // 中心座標を返す
  getCenter(s) {
    return { x: s.x + s.w / 2, y: s.y + s.h / 2 };
  },

  // 移動
  move(s, dx, dy) {
    s.x += dx;
    s.y += dy;
  },

  // リサイズ（handle: 'se'|'sw'|'ne'|'nw'|'n'|'s'|'e'|'w'）
  resize(s, handle, start, nx, ny, nw, nh) {
    s.x = nx; s.y = ny;
    s.w = Math.max(4, nw);
    s.h = Math.max(4, nh);
  },

  // SVG 書き出し用
  toSVG(s) {
    return `<rect x="${s.x}" y="${s.y}" width="${s.w}" height="${s.h}"
              fill="none" stroke="${s.color}" stroke-width="${s.sw || 2}"/>`;
  },

  // HTML プレビュー用のコード文字列
  previewDrawCode: `
    ctx.strokeRect(s.x, s.y, s.w, s.h);
  `
});
コールバック必須役割
draw(ctx, s)必須Canvas 2Dコンテキストに図形を描画する。必ず ctx.save() / ctx.restore() で挟むこと
getBounds(s)任意選択枠・当たり判定に使うバウンディングボックスを返す
getCenter(s)任意回転の中心などに使う中心座標を返す
move(s, dx, dy)任意ドラッグ移動時に図形の座標を更新する
resize(s, handle, ...)任意選択枠のハンドルドラッグでサイズ変更する
toSVG(s)任意SVG書き出し機能で使われるマークアップ文字列を返す
previewDrawCode任意HTML/JS書き出し時に埋め込まれる描画コードの文字列

図形操作

addShape(shape)

キャンバスに新しい図形を追加します。アクティブなレイヤーに自動で配置され、レイヤーパネルにも表示されます。type には registerShapeType で登録したキー、もしくは組み込みの図形タイプを指定します。

api.addShape({
  type: "my_shape",      // registerShapeType で登録した type
  name: "マイ図形",      // レイヤーパネルの表示名
  x: 100, y: 100,
  w: 120, h: 80,
  // color / sw / opa / dash / fill / rot は省略時にエディタ設定値が使われる
});

Shapeの共通フィールド:

フィールド型説明
typestring図形タイプ
namestringレイヤーパネル表示名
colorstring色 (CSS color)
swnumber線の太さ
opanumber不透明度 (0–100)
dashstring破線パターン例: "5,3"
fillboolean塗りつぶし
rotnumber回転角度 (度)
hiddenboolean個別非表示
enginestring"canvas2d" (省略時) または "threejs"
layerIdstring配置レイヤー (省略時はアクティブレイヤー)
addShape で追加した図形は自動的にキーフレームアニメーションの対象になります。draw(ctx, s) に渡される s.x / s.y などはすでに補間済みの値なので、アニメーション対応のために特別なコードを書く必要はありません。

getSelected()

現在選択中の図形オブジェクトを返します。未選択時は null が返るため、必ずnullチェックしてから使ってください。

const s = api.getSelected();
if (s) console.log(s.type, s.x, s.y);

setSelectedPatch(patch)

選択中の図形のプロパティを部分的に更新し、左側のプロパティパネルの表示も自動的に同期します。カスタムUIのスライダーなどと組み合わせてよく使われます。

api.setSelectedPatch({ color: "#ff0000", sw: 4 });

requestRender() / redraw()

キャンバスの再描画を要求します。図形データを直接書き換えた後など、画面に即座に反映させたいときに api.redraw() を呼びます。

api.redraw();

UI拡張

registerUI(ui)

任意のパネルエリアにカスタムUIを追加します。簡単なHTML文字列を渡すだけでよく、より高度なDOM操作が必要な場合は render / onMount コールバックを使います。

api.registerUI({
  id: "my_ui",
  position: "right",     // "top" | "right" | "bottom" | "left" | "editor-top"
  title: "マイ設定",     // オプション
  html: `<button onclick="doSomething()">実行</button>`,

  // render / onMount で DOM を直接操作することもできる
  render(wrap, api) { ... },
  onMount(wrap, api) { ... }
});

position によって挿入先が変わります。ツールのオプションパネルには "right"、テキストエディタ画面に出したい場合は "editor-top" を使うのが一般的です。

registerFileMenuItem(item)

上部の「File」メニューに独自の項目を追加します。書き出し・読み込みなど、ファイル操作系の機能をMODから提供したいときに使います。

api.registerFileMenuItem({
  id: "my_export",
  label: "カスタム書き出し",
  icon: "ti-download",   // Tabler Icons クラス名（省略可）
  onClick(api) {
    // 書き出し処理
  }
});

レイヤー

createLayer(options)

新しいレイヤーを作成してレイヤーパネルに追加します。作成したレイヤーは自動的にアクティブレイヤーになります。フォルダ型のレイヤーを作って、その中に子レイヤーをまとめることもできます。

const layer = api.createLayer({
  name: "WebGL Effects",    // 省略時 "New Layer"
  type: "normal",           // "normal"（省略時）または "folder"
  parentId: null,           // フォルダ内に入れる場合は親フォルダの id
  color: "#8b5cf6",         // レイヤーパネルのカラードット（省略時なし）
  visible: true,            // 省略時 true
  locked: false,            // 省略時 false
  opacity: 1,               // 0.0〜1.0（省略時 1）
  blendMode: "source-over"  // CSS globalCompositeOperation（省略時 "source-over"）
});
// → 作成したレイヤーオブジェクトを返す

典型的な使い方（addObjectと組み合わせる）:

const layer = api.createLayer({ name: "Particle Layer", color: "#ff9900" });

api.addObject({
  engine: "threejs",
  type: "particle",
  name: "パーティクル",
  layerId: layer.id       // 作成したレイヤーに配置
});

フォルダを作成してレイヤーを入れる:

const folder = api.createLayer({ name: "WebGL Group", type: "folder" });
const child  = api.createLayer({ name: "Effects",     parentId: folder.id });
type: "folder" のレイヤーは addShape / addObject の対象になりません。図形の追加先は getDrawableActiveLayerId() が自動的に子レイヤーへ振り分けます。

getLayers / isLayerVisible / isLayerLocked

現在のレイヤー構成を読み取るためのAPI群です。フォルダの表示/ロック状態が子レイヤーに継承されるかどうかも考慮した結果を返します。

// 全レイヤーを取得
const layers = api.getLayers();
// 例: [ { id, name, type, visible, locked, opacity, ... }, ... ]

// レイヤー状態の確認（フォルダの継承も考慮）
api.isLayerVisible("layer-1");  // → boolean
api.isLayerLocked("layer-1");   // → boolean

Three.js連携

addObject(obj) / removeObject(id) / updateObject(id, patch)

Three.jsなど、Canvas 2D以外のレンダリングエンジンを使うオブジェクトを shapes[] に登録するためのAPIです。登録することで、通常の図形と同じくレイヤーパネルでの表示/非表示やロックの対象になります。

const obj = api.addObject({
  type: "my_3d_object",
  engine: "threejs",          // 省略時 "threejs"
  name: "3Dオブジェクト",
  color: "#3B8AE6",
  sw: 1, opa: 100, dash: "0", fill: false
});

api.updateObject(obj.id, { name: "名前変更" });
api.removeObject(obj.id);

registerThreeRenderer(callbacks)

Three.jsレンダラー側がレイヤーの状態変化（表示切替・ロック・オブジェクトの個別非表示）を受け取るためのコールバックを登録します。addObject と組み合わせて、2DレイヤーパネルとWebGLシーンの状態を同期させるために使います。

api.registerThreeRenderer({
  onLayerVisibility(layerId, visible) { ... },
  onLayerLock(layerId, locked)        { ... },
  onObjectHidden(objectId, hidden, shape) { ... }
});

ユーティリティ

toast / setStatus / getBounds / getCenter / getSceneSnapshot

通知の表示や、図形の座標計算、シーン全体の情報取得など、細かい場面で使う補助的なAPIです。

api.toast("ti-check", "完了しました");   // トースト通知
api.setStatus("処理中...");              // ステータスバー

const bounds = api.getBounds(shape);    // → { x, y, w, h }
const center = api.getCenter(shape);    // → { x, y }

const snapshot = api.getSceneSnapshot();
// → { width, height, bg, totalDur, looping, fps, shapes[] }

getObjectsByLayer(layerId) / getObjectsByEngine(engine)

特定のレイヤーやエンジンに属する図形だけを絞り込んで取得します。Three.jsオブジェクトだけをまとめて処理したいときなどに便利です。

// 特定レイヤーの全オブジェクト（エンジン問わず）
api.getObjectsByLayer("layer-1");

// 特定エンジンの全オブジェクト
api.getObjectsByEngine("threejs");

よくあるパターン

実際のMOD開発でよく出てくる実装パターンを、コード例つきで紹介します。

プロパティパネルと連動する

選択中の図形のプロパティをカスタムUIから変更したいときは、setSelectedPatch をイベントリスナー内で呼びます。

document.getElementById("my-slider").addEventListener("input", function () {
  api.setSelectedPatch({ myProp: Number(this.value) });
});

カスタムUIを右パネルに追加する

registerUI の onMount でDOM要素を取得し、イベントを紐づけます。api.getSelected() と組み合わせて、選択中の図形の種類をチェックしてから更新するのが安全です。

api.registerUI({
  id: "my_ui",
  position: "right",
  title: "オプション",
  html: `<label>サイズ <input id="my-size" type="range" min="10" max="200" value="60"></label>`,
  onMount(wrap, api) {
    wrap.querySelector("#my-size").addEventListener("input", function () {
      const s = api.getSelected();
      if (s && s.type === "my_shape") api.setSelectedPatch({ r: Number(this.value) });
    });
  }
});

ファイルメニューに書き出し項目を追加する

getSceneSnapshot() でシーン全体を取得し、任意の形式に変換して書き出せます。

api.registerFileMenuItem({
  id: "my_export",
  label: "JSON で書き出し",
  icon: "ti-file-export",
  onClick(api) {
    const scene = api.getSceneSnapshot();
    const blob = new Blob([JSON.stringify(scene, null, 2)], { type: "application/json" });
    const url = URL.createObjectURL(blob);
    const a = document.createElement("a");
    a.href = url; a.download = "scene.json"; a.click();
    URL.revokeObjectURL(url);
    api.toast("ti-check", "書き出し完了");
  }
});

マウスでキャンバスに図形を配置する

カスタムツールを作る場合の定番パターンです。mousedown で開始点を記録し、mouseup で addShape を呼びます。他のツールと競合しないよう、e.stopImmediatePropagation() を必ず入れてください。

const cv = document.getElementById("cv");
let drawing = false, startX = 0, startY = 0;

function canvasPos(e) {
  const r = cv.getBoundingClientRect();
  return { x: e.clientX - r.left, y: e.clientY - r.top };
}

cv.addEventListener("mousedown", function (e) {
  if (api.activeModTool?.id !== "my_tool") return;
  e.preventDefault();
  e.stopImmediatePropagation();
  const p = canvasPos(e);
  startX = p.x; startY = p.y;
  drawing = true;
}, true);

cv.addEventListener("mouseup", function (e) {
  if (!drawing || api.activeModTool?.id !== "my_tool") return;
  e.preventDefault();
  e.stopImmediatePropagation();
  drawing = false;

  const p = canvasPos(e);
  const x = Math.min(startX, p.x);
  const y = Math.min(startY, p.y);
  const w = Math.max(10, Math.abs(p.x - startX));
  const h = Math.max(10, Math.abs(p.y - startY));

  api.addShape({ type: "my_shape", name: "My Shape", x, y, w, h });

  // ツールを解除して選択ツールに戻る
  api.activeModTool = null;
  document.querySelectorAll(".rp-btn[data-mod-tool]")
          .forEach(b => b.classList.remove("active"));
  if (window.setTool) window.setTool("select");
}, true);

アニメーションに対応する

addShape で追加した図形は自動的にキーフレームアニメーションの対象になります。draw(ctx, s) では s.x, s.y などが補間済みの値で渡されるため、特別な対応は不要です。

チェックリスト

MODを公開・共有する前に、次の項目を確認してください。

  • mod.json の id がフォルダ名と一致している
  • main.js が即時関数 (function(){ ... })() で囲まれている
  • api の存在確認（if (!api) return;）がある
  • registerMod() が最初に呼ばれている
  • draw() 内で ctx.save() / ctx.restore() をしている
  • getBounds() が正しいバウンディングボックスを返している
  • マウスイベントで e.preventDefault() + e.stopImmediatePropagation() をしている
